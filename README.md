# 🤖 Бот «Личный Ассистент Дня»

Полная реализация плана: сопровождает весь день — от утреннего брифинга до вечернего итога.

Что умеет:
- ✅ Задачи и напоминания свободной фразой: «напомни завтра в 9 позвонить маме»
- 📸 Фото холодильника → список продуктов + 2–3 рецепта
- 🍳 Список продуктов текстом → рецепты + кнопка «докупить недостающее»
- 🛒 Единый список покупок: «купить молоко» и недостающие ингредиенты из рецептов — в одном списке; можно вести вдвоём (/share + /join)
- 💸 Учёт расходов одной строкой: «кофе 250» → итоги за день/неделю (/spent)
- 🔁 Трекер привычек с напоминаниями (/habit Пить воду 10:00)
- 🧊 Виртуальный холодильник и сроки годности: «в холодильнике курица до 25.07» → напомнит и предложит рецепт «спасения»
- 📅 Меню на неделю + общий список покупок (/menu)
- ☀️ Утренний брифинг: погода + задачи + новости + «скоро испортится»
- 🌙 Вечерний итог: выполненное, перенос на завтра, оценка дня 1–10, дневник настроения (/mood)
- 📱 Мини-апп внутри Telegram: задачи, покупки, холодильник, привычки и расходы на одном экране (/app)

---

## Шаг 1. Получи ключи (2 обязательных + 1 по желанию)

1. **Токен бота** — в Telegram найди `@BotFather` → `/newbot` → придумай имя → скопируй токен вида `123456:ABC-DEF...`
2. **Ключ Groq API** — [console.groq.com/keys](https://console.groq.com/keys) → войди через Google-аккаунт → «Create API Key». Бесплатно, карта не нужна. Бот использует модель `llama-3.3-70b-versatile` для текста и `llama-4-scout` для фото
3. *(необязательно)* **Ключ погоды** — [openweathermap.org](https://openweathermap.org/api) → бесплатная регистрация → API key. Без него всё работает, просто в брифинге не будет погоды.

## Шаг 2. Настрой .env

Скопируй `.env.example` в файл с именем `.env` и вставь свои ключи:

```
BOT_TOKEN=123456:ABC-DEF...
GROQ_API_KEY=gsk_...
OPENWEATHER_API_KEY=...
```

## Шаг 3. Запуск

### Локально (проверить на своём ПК)

```bash
pip install -r requirements.txt
python3 main.py
```

### На хостинге (ядро node.js + python3)

1. Залей все файлы папки на сервер (включая заполненный `.env`)
2. В консоли сервера: `pip install -r requirements.txt` (или `pip3`)
3. Команда запуска: `python3 main.py` (или через `index.js`, если панель стартует с Node)

Бот работает в режиме polling — никаких портов, доменов и webhook настраивать не нужно.

## Шаг 4. Проверь

1. Напиши боту `/start`
2. Настрой: `/city Москва`, `/time 07:30`, `/eveningtime 21:00`
3. Попробуй:
   - «напомни через 2 минуты попить воды» → придёт напоминание
   - «курица, картошка, сметана» → рецепты + кнопка «добавить недостающее в список»
   - фото холодильника → распознает продукты и предложит рецепты
   - «кофе 250» → запишет трату
   - «купить молоко и хлеб» → добавит в список покупок
   - «в холодильнике курица до 25.07» → запомнит срок годности
   - `/digest` и `/evening` → брифинг и итог дня прямо сейчас

## Мини-апп (по желанию)

Внутри бота есть Telegram Mini App — экран с задачами, покупками, холодильником, привычками и расходами. Веб-сервер поднимается автоматически вместе с ботом (порт из `WEB_PORT`, по умолчанию 8080).

Чтобы кнопка мини-аппа появилась, нужен **публичный HTTPS-адрес** сервера:

1. Если хостинг даёт домен (например, `https://mybot.example.com`) — укажи его в `.env`: `WEBAPP_URL=https://mybot.example.com`
2. Перезапусти бота — в меню чата появится кнопка «Мини-апп», также работает команда `/app`
3. Локально без домена можно проверить через туннель (например, `ngrok http 8080` → вставь выданный https-адрес в `WEBAPP_URL`)

Без `WEBAPP_URL` бот работает как обычно, просто кнопка мини-аппа не показывается.

## Команды бота

| Команда | Что делает |
| --- | --- |
| `/start`, `/help` | приветствие и справка |
| `/city Москва` | город для погоды |
| `/time 07:30` | время утреннего брифинга |
| `/eveningtime 21:00` | время вечернего итога |
| `/tasks`, `/done 3` | задачи: список и закрытие |
| `/list`, `/buy ...`, `/bought 2`, `/clearlist` | список покупок |
| `/share`, `/join КОД`, `/unjoin` | общий список покупок на двоих |
| `/spent` | расходы за день и неделю |
| `/habit Пить воду 10:00`, `/habits`, `/delhabit 2` | привычки |
| `/fridge` | виртуальный холодильник (убрать: `/fridge 2`, очистить: `/fridge clear`) |
| `/menu` | меню ужинов на неделю + список покупок |
| `/digest`, `/evening` | брифинг / итог дня прямо сейчас |
| `/mood` | дневник настроения |
| `/app` | открыть мини-апп (нужен WEBAPP_URL в `.env`) |

## Устройство

- `main.py` — точка входа (запуск бота и фонового цикла)
- `config.py` — все настройки из `.env`
- `db.py` — SQLite: схема и функции (пользователи, задачи, покупки, расходы, привычки, холодильник, настроение)
- `llm.py` — клиент Groq API (OpenAI-совместимый) и промпты
- `services.py` — погода, новости, сборка утреннего/вечернего дайджестов
- `handlers/` — обработчики команд по блокам (basic, tasks, shopping, kitchen, life, text_router)
- `scheduler.py` — фоновый цикл: напоминания, рассылки, сроки годности
- `webapp.py` — встроенный веб-сервер мини-аппа (aiohttp) + REST API с проверкой подписи Telegram
- `assets/webapp.html` — интерфейс мини-аппа (один файл, без сборки)
- `index.js` — запускатель для хостингов с ядром node.js + python3
- `assistant.db` — база SQLite, создаётся автоматически при первом запуске

## Частые проблемы

- **Бот не отвечает** — проверь BOT_TOKEN в `.env` и что бот запущен (в логах есть «Бот запущен»)
- **Ошибка 401/403 от LLM** — неверный GROQ_API_KEY — создай новый на console.groq.com/keys
- **Ошибка 429 от LLM** — исчерпан бесплатный лимит Groq на минуту/день; бот сам переключается на запасную модель, но при частых ошибках подожди минуту
- **Хочу другую модель** — в `.env`: `LLM_MODEL=llama-3.1-8b-instant` (быстрее) или любой OpenAI-совместимый API через `LLM_BASE_URL` + `LLM_API_KEY`
- **Нет погоды** — ключ OpenWeatherMap активируется до пары часов после регистрации
- **Не нужны новости в брифинге** — поставь в `.env` пустое значение: `NEWS_RSS_URL=`
- **Напоминания приходят не в то время** — проверь TZ_NAME в `.env` (например, Europe/Moscow)
- **Мини-апп не открывается** — Telegram требует HTTPS: проверь, что WEBAPP_URL начинается с `https://` и адрес доступен снаружи
